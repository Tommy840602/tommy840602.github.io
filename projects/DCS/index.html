分散式控制系統（Web-Based DCS）

針對 Czochralski 長晶爐打造的 Web-Based DCS 模擬平台，整合分散式 PID Controller、Primary／Hot-Standby 備援、MQTT Sparkplug B、OPC UA、Spring Boot 與 Vue 3 即時操作站。

System Architecture

Highlights

建立六組獨立 PID 控制迴路，涵蓋晶棒直徑、加熱溫度、氬氣流量、爐壓、坩堝位置與晶種提升速度。

每組控制迴路配置 Primary 與 Hot-Standby Controller，透過 MQTT Retained Lease 決定 Active Controller。

控制節點以 100 ms Scan Cycle 獨立執行，即使 Supervisory Gateway 暫時中斷，仍可持續本地控制掃描。

使用 MQTT QoS 1、Sparkplug B Protobuf、Birth／Death Certificate 與 Metric Alias 建立工業設備通訊模型。

使用 Eclipse Milo 提供 OPC UA Server，讓外部 Client 以標準工業協定讀取製程與控制狀態。

透過 HAProxy、Mosquitto Primary／Standby 實作 MQTT Broker Failover。

使用 Vue 3 與 WebSocket 建立即時 Operator Station，顯示 SP、PV、PID Output、Controller 狀態與告警資訊。

整合 PostgreSQL、Redis、RBAC、Audit Trail、工程腳本審查、版本控管與回滾流程。

Engineering Focus

此專案模擬 DCS 中控制層與監控層的責任分離。PID 控制由獨立 Controller Node 執行，Gateway 僅負責監控、認證、命令路由、OPC UA 與 WebSocket Telemetry，避免網站或資料庫成為即時控制路徑的一部分。

系統亦透過 Hot-Standby Controller、Broker Failover、通訊品質檢查及安全輸出行為，展示分散式控制系統的容錯與高可用設計。
