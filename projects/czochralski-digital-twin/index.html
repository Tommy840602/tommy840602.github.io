System Architecture

Highlights

建立從設備資料、MQTT、Kafka、Flink 到前端視覺化的端到端即時資料管線。

使用 INGOT_NO 作為 Kafka Partition Key，維持同一晶棒事件的處理順序，並降低多爐並行資料互相干擾的風險。

透過 Flink Stateful Processing 與 CEP 偵測製程異常，再將告警事件交由獨立 Alarm Service 處理與通知。

依資料特性分別使用 TimescaleDB、MongoDB、Redis 與 Elasticsearch，處理時序資料、原始事件、即時狀態及集中式日誌。

使用 Vue、Three.js、ECharts 與 WebSocket 建立即時 3D 爐況、製程 KPI、歷史趨勢及告警介面。

整合 JWT、RBAC、Prometheus、Grafana、ELK、Docker Compose、Caddy HTTPS 與正式環境部署。

Engineering Focus

此專案展示如何將工業設備資料轉換成可擴充、可追蹤且可即時呈現的事件流系統。設計重點包含設備與後端解耦、事件順序、串流異常偵測、多資料庫分工、微服務安全邊界及正式環境可觀測性。
