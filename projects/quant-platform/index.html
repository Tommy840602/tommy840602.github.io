量化交易平台（Quant Platform）

一套可實際運行的事件驅動量化交易平台，整合 Binance 即時行情、技術分析、策略引擎、OMS（Order Management System）、風險控管、回測、帳戶管理與通知系統，完成從市場資料到成交回報的完整交易閉環。

System Architecture

Highlights

市場行情透過 Binance WebSocket 接收後發布至 Kafka，由 Strategy Service 消費事件並產生交易訊號，再交由 Order Execution Service 完成交易流程。

實作完整 OMS，包括冪等處理、交易所 Filter 校正、HMAC 簽名下單、成交回報、撤單、訂單查詢與交易生命週期管理。

提供全域 Kill Switch、單筆名目金額限制、餘額檢查、最大回撤熔斷及 Testnet／Mainnet／Dry-run 多環境切換，降低正式交易風險。

結合 Java 事件驅動回測與 Python 向量化分析，共用策略介面，降低回測與實盤交易邏輯不一致的問題。

提供即時 K 線、技術指標、交易訊號、部位、資產淨值、資金曲線與策略狀態，支援多交易對即時監控。

整合 Prometheus、Grafana 與 Slack 通知，即時監控交易系統、成交狀態、風控事件及服務健康狀態。

Engineering Focus

本專案著重於建立可部署的量化交易平台，而非單純策略研究。

系統透過事件驅動架構串接市場資料、策略、訂單、風控與帳戶管理，並實作冪等控制、交易所規則校正、正式資金保護、微服務解耦及可觀測性，展示金融交易系統在高可靠性、高一致性及高可維護性的工程設計。
